## Summary

Define all enum types required for live trading: `ExecutionMode`, `SessionStatus`, `OrderStatus`, `OrderSide`, and `OrderType`. These enums provide type-safe state management for the live trading domain.

## Implementation Details

### File Location
`src/domain/live_trading/enums.py`

### Enums to Define

```python
from enum import Enum

class ExecutionMode(Enum):
    """Execution mode for live trading sessions"""
    MANUAL = "manual"      # User confirms each trade
    SEMI_AUTO = "semi_auto"  # User confirms, then auto-execute for N seconds
    AUTO = "auto"          # Fully automatic (requires explicit consent)

class SessionStatus(Enum):
    """Status of a live trading session"""
    CREATED = "created"    # Session created but not started
    RUNNING = "running"    # Actively trading
    PAUSED = "paused"      # Temporarily paused
    STOPPED = "stopped"    # Permanently stopped
    ERROR = "error"        # Error state

class OrderStatus(Enum):
    """Status of a live order"""
    PENDING = "pending"      # Awaiting confirmation
    SUBMITTED = "submitted"  # Sent to broker
    PARTIAL = "partial"      # Partially filled
    FILLED = "filled"        # Fully filled
    CANCELLED = "cancelled"  # Cancelled by user
    REJECTED = "rejected"    # Rejected by broker
    FAILED = "failed"        # Submission failed

class OrderSide(Enum):
    """Side of an order"""
    BUY = "buy"
    SELL = "sell"

class OrderType(Enum):
    """Type of order"""
    MARKET = "market"
    LIMIT = "limit"
```
