## Executive Summary

Create the main `PaperTradingView` widget that orchestrates the paper trading UI. Includes session status display, control buttons (start/pause/resume/stop), and layout for child widgets (PortfolioWidget, SignalsWidget, SessionStatsWidget). Connects to the paper trading engine for data updates.

## Execution Flow

```
1. Create main view class
   → Inherit from QWidget
   → Define signals for session events
2. Setup UI layout
   → Header with session info and status
   → Control buttons row
   → Main content area with child widgets
3. Implement session controls
   → Start/Pause/Resume/Stop buttons
   → Connect to engine methods
4. Connect to engine signals
   → Update on portfolio changes
   → Update on new signals
   → Update stats periodically
   → Return: SUCCESS with view working
```
