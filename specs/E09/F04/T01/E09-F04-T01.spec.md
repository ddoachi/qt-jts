## Executive Summary

Create the main StrategyView widget that serves as the container for Strategy Builder. Features a split layout with strategy list on the left and editor on the right, plus a toolbar with action buttons for creating, saving, and running strategies.

## Execution Flow

```
1. Create StrategyView(QWidget)
   → Main container for Strategy Builder tab
2. Setup split layout
   → QSplitter with list and editor panels
   → Configurable split ratio
3. Add toolbar
   → New, Save, Delete buttons
   → Backtest and Paper Trade buttons
4. Connect signals
   → List selection → load in editor
   → Save button → call use case
   → Return: SUCCESS with main view ready
```

## User Stories

### Primary User Story
**As a** trader
**I want to** access the Strategy Builder interface
**So that** I can create and manage strategies

## Acceptance Scenarios

### Scenario 1: View Layout
**Given** StrategyView is displayed
**Then** left panel shows strategy list
**And** right panel shows strategy editor

### Scenario 2: Select Strategy
**Given** strategy list with items
**When** user selects a strategy
**Then** editor loads selected strategy

### Scenario 3: Create New Strategy
**Given** StrategyView with toolbar
**When** user clicks "New"
**Then** editor clears for new strategy
