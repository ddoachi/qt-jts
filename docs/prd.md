## 1. Executive Summary

Joohan Trading System (JTS) is a comprehensive trading workstation that enables traders to collect market data, discover profitable patterns, backtest strategies, and execute trades across multiple brokers. The system supports multiple broker accounts with intelligent rate limiting, ensuring reliable data collection and order execution.

---

## 2. Problem Statement

Traders face several challenges:

1. **Data Acquisition**: Collecting years of historical data from brokers is tedious and error-prone
2. **Multi-Account Complexity**: Managing multiple broker accounts with different rate limits is difficult
3. **Information Overload**: Monitoring hundreds of stocks manually is impossible
4. **Pattern Recognition**: Identifying recurring profitable patterns requires analyzing years of data
5. **Strategy Validation**: Testing trading ideas without risking real money is difficult
6. **Rule Formulation**: Converting intuition into systematic, repeatable trading rules is complex

#### 5.0.3 Rate Limiter

Each broker has its own rate limiter that:
- Tracks requests per second across all operations
- Queues requests when limit is reached
- Prioritizes trading orders over data requests
- Shows real-time usage indicator

#### 5.0.4 Account Selection

For each operation, user selects which account to use:
- **Data Collection**: Select broker (account not required for market data)
- **Paper Trading**: Virtual account (no real broker needed)
- **Live Trading**: Select specific broker + account

### 5.2 Market Scanner

**Purpose**: Find stocks matching custom-defined patterns across the entire market.

#### 5.2.1 Scan Formula Definition

Users can create reusable scan formulas using a simple expression language:

**Example Formulas**:
```
# Volume Surge: Find stocks with unusual volume
volume > average(volume, 20) * 2.5

# Golden Cross: Moving average crossover
sma(close, 50) crosses_above sma(close, 200)

# Oversold Bounce: RSI reversal
rsi(14) < 30 AND close > open

# Breakout: Price breaks 52-week high
close > highest(high, 252)
```

**Formula Features**:
- Parameterizable (e.g., `:period`, `:threshold`)
- Saveable and shareable
- Tag-based organization
- Version history

#### 5.2.2 Scan Execution

| Setting | Options |
|---------|---------|
| Data Source | Local (downloaded), Live (from broker) |

#### 5.8.2 Prerequisites

Before live trading:
- [ ] Strategy backtested with acceptable metrics
- [ ] Paper trading completed for minimum period
- [ ] Risk parameters configured
- [ ] Broker account connected and verified

#### 5.8.3 Risk Controls

| Control | Description |
|---------|-------------|
| Max Daily Loss | Stop trading if daily loss exceeds limit |
| Max Position Size | Limit per-trade capital allocation |
| Max Open Positions | Limit concurrent trades |
| Trading Hours | Only trade during specified hours |

#### 5.8.4 Signal Execution

When a signal is generated:
1. Display notification with signal details
2. User confirms or rejects execution
3. Order submitted to selected broker account
4. Confirmation displayed with order ID

**Optional**: Auto-execution mode (requires explicit user consent)
